# TypeScript Backend Patterns You Should Know

TypeScript has become the language of choice for many backend projects. Let's explore essential design patterns that make your backend code more maintainable.

## Repository Pattern

The repository pattern abstracts data access logic, making it easier to test and swap out data sources:

```typescript
interface UserRepository {
  findById(id: string): Promise<User | null>;
  create(data: CreateUserDto): Promise<User>;
  update(id: string, data: UpdateUserDto): Promise<User>;
  delete(id: string): Promise<void>;
}
```

## Dependency Injection

Rather than hardcoding dependencies, inject them:

```typescript
class UserService {
  constructor(
    private readonly userRepo: UserRepository,
    private readonly emailService: EmailService,
    private readonly logger: Logger,
  ) {}

  async createUser(data: CreateUserDto) {
    const user = await this.userRepo.create(data);
    await this.emailService.sendWelcome(user.email);
    this.logger.info(`User created: ${user.id}`);
    return user;
  }
}
```

## Result Pattern

Instead of throwing exceptions for expected failures, use a Result type:

```typescript
type Result<T, E = Error> = { ok: true; value: T } | { ok: false; error: E };
```

## Conclusion

These patterns help build backends that are testable, maintainable, and scalable. Adopt them incrementally and see the difference they make.
